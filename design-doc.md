# MindPulse 产品设计文档

> 让读过的东西，真正变成你的。

## 1. 产品概述

### 1.1 产品定位

MindPulse 是一款面向职场人的 iOS App，通过 AI 自动将用户日常阅读的内容转化为间隔重复卡片，并在每次复习后以极简方式记录个人状态，最终通过数据洞察揭示知识学习与个人状态之间的关联。

### 1.2 核心价值主张

- **零门槛制卡**：扔进一篇文章，AI 自动生成复习卡片
- **两分钟复习**：通勤路上滑几张卡，碎片时间就够
- **状态感知**：不是打卡工具，而是帮你发现"学什么让你状态更好"

### 1.3 目标用户

**主画像：城市职场人，25-40 岁**

| 特征 | 描述 |
|------|------|
| 信息习惯 | 每天阅读大量文章、播客、公众号，但留存率低 |
| 痛点 | "看过≠记住"，收藏夹吃灰 |
| 时间特征 | 碎片化，通勤 30-60 分钟，午休 30 分钟 |
| 心态 | 有成长焦虑，但讨厌打卡式的自律压迫感 |
| 设备 | iPhone 为主力机 |

```
│ Share Ext│ 记得/忘了  │  AI 洞察       │
└─────────┴──────────┴────────────────┘
```

### 2.2 功能详细说明

#### F1：内容输入

**目标**：让用户在 5 秒内把内容扔进来。

| 输入方式 | 说明 | 优先级 |
|---------|------|--------|
| 粘贴文本 | App 内文本框，粘贴文章片段或笔记 | P0 (Phase 1) |
| 输入 URL | 粘贴链接，后台抓取网页正文 | P0 (Phase 1) |
| Share Extension | 任意 App 点"分享"直接送入 | P1 (Phase 2) |
| 剪贴板识别 | 打开 App 时检测剪贴板中的 URL/文本，弹出提示 | P1 (Phase 2) |

**输入后的处理流程：**

```
用户输入内容
    ↓
[URL?] → 是 → 抓取网页正文(Readability)
    ↓ 否
  原始文本
    ↓
  调用 Claude API
    ↓
  返回 3~5 张卡片 (问题 + 答案 + 原文引用)
    ↓
  存入本地数据库
    ↓
  显示卡片预览 → 用户可删除不想要的卡片
```

**失败态交互（必须覆盖）：**

1. URL 抓取失败：显示失败原因 + `重试` + `改为手动粘贴正文`
2. AI 请求超时/网络异常：显示 `重试` + `稍后处理（保存到草稿）`
3. AI 返回格式异常（JSON 解析失败）：自动重试 1 次；仍失败则降级为 `手动建卡`
4. 任何失败态都不静默：必须有明确提示和下一步按钮

**AI 生成卡片的 Prompt 设计：**

```
你是一个知识提炼专家。请阅读以下内容，提取 3-5 个最有价值的知识点，
生成间隔重复卡片。

要求：
1. 每张卡片包含：问题（正面）、答案（背面）、原文引用（一句话）
2. 问题要具体，避免"是什么"这类空泛问题
3. 优先提取：反直觉的观点、可操作的方法论、关键数据
4. 答案控制在 2-3 句话内
5. 用中文输出

输出 JSON 格式：
[
  {
    "question": "...",
    "answer": "...",
    "source_quote": "..."
  }
]

以下是内容：
---
{content}
---
```

#### F2：卡片复习

**目标**：每天 2-3 分钟，无压力地复习。

**交互设计：**

```
┌────────────────────────┐
│                        │
│   [知识领域标签]         │
│                        │
│   问题内容显示在这里      │
│   支持多行文本           │
│                        │
│                        │
│       点击翻转 ↓        │
│                        │
│                        │
│  ← 忘了        记得 →   │
│                        │
│   ━━━━━━━○━━━━━━━━━━    │
│     3/8 张              │
│                        │
└────────────────────────┘
```

- **点击卡片**：翻转显示答案 + 原文引用
- **右滑 / 点击"记得"**：标记为记得，进入下一张
- **左滑 / 点击"忘了"**：标记为忘了，进入下一张
- **误操作保护**：提交后底部显示 5 秒 `撤回上一张`（仅可撤回一次）
- **底部进度条**：显示今天的复习进度

**间隔重复算法（SM-2）：**

```
每张卡片维护以下状态：
- repetition: 连续正确次数
- ease_factor: 难度因子 (初始 2.5)
- interval: 下次复习间隔天数

用户回答后更新：

if 回答 == "记得":
    if repetition == 0:
        interval = 1
    elif repetition == 1:
        interval = 6
    else:
        interval = round(interval * ease_factor)
    repetition += 1
    ease_factor = max(1.3, ease_factor + 0.1)

if 回答 == "忘了":
    repetition = 0
    interval = 1
    ease_factor = max(1.3, ease_factor - 0.2)

next_review_date = today + interval 天
```

**每日卡片选择逻辑：**

```
1. 选出所有 next_review_date <= 今天 的卡片
2. 按优先级排序：过期天数多的优先，ease_factor 低的优先
3. 上限 8 张（避免负担感）
4. 如果到期卡片不足 3 张，从最新添加且未复习的卡片中补充
```

#### F3：状态记录

**目标**：复习完成后，用 3 秒记录当下状态。

**交互设计：**

```
┌────────────────────────┐
│                        │
│     今天复习了 5 张 ✓   │
│     记住了 4 张 (80%)   │
│                        │
│   ─── 今日状态 ───      │
│                        │
│   🪫━━━━━━━━━━🔋       │
│       能量值: 7/10      │
│                        │
│   今天的关键词(可选):    │
│   ┌──────────────────┐ │
│   │ 例如：充实、疲惫   │ │
│   └──────────────────┘ │
│                        │
│      [ 完成 ]          │
│                        │
└────────────────────────┘
```

- **能量滑块**：0-10 的连续滑块，默认值为上次的值
- **关键词**：可选输入，支持自定义，也提供快捷标签（充实/疲惫/焦虑/平静/兴奋）
- **提交规则（统一）**：点击`完成`后立即存储，并在底部显示 5 秒`撤回`；超时后不可再编辑

#### F4：洞察报告

**目标**：每周一份报告，让用户看到知识和状态的关系。

**周报内容：**

```

│   Mon  ████████░░ 8        │

│   Wed  █████░░░░░ 5        │

│   Fri  ███████░░░ 7        │
```

## 3. 信息架构

### 3.1 页面结构

```
TabBar 底部导航（3 个 Tab）
├── 📚 复习 (默认首页)
│   ├── 今日卡片复习
│   ├── 复习完成 → 状态记录
│   └── 无卡片时 → 空状态引导添加内容
├── ➕ 添加
│   ├── 粘贴文本
│   ├── 输入 URL
│   └── AI 生成中 → 卡片预览 → 确认保存
└── 📊 洞察
    ├── 本周概览
    ├── 能量趋势图
    ├── 知识领域分布
    └── AI 洞察卡片
```

### 3.2 核心用户旅程

**旅程 A：首次使用**

```
下载 App
  → 欢迎页 (3 页简短引导，不超过 30 秒)
    1. "把你读到的好内容扔进来"
    2. "AI 帮你提炼成记忆卡片"
    3. "每天 2 分钟，让知识真正留下"
  → 先体验 2 张示例卡（立即感知核心价值）
  → 引导添加第一篇内容 (粘贴文本或 URL)
  → AI 生成卡片 → 预览
  → 体验第一次真实复习 (2-3 张卡片)
  → 状态记录
  → "明天同一时间来复习吧" → 请求通知权限
```

**旅程 B：日常复习（核心循环）**

```
收到推送通知："今天有 5 张卡片等你复习"
  → 打开 App → 直接进入复习
  → 滑动完成所有卡片 (~2 分钟)
  → 弹出状态记录 → 滑动能量条 → 完成
  → 回到首页，显示"今日已完成 ✓"
```

**旅程 C：添加新内容**

```
在微信/浏览器/播客App看到好内容
  → 复制链接或文字
  → 打开 MindPulse (或使用 Share Extension)
  → 粘贴内容 → 点击"生成卡片"
  → 等待 3-5 秒
  → [成功] 预览卡片 → 删除不想要的 → 确认保存
  → [失败] 错误提示 → 重试 或 改为手动粘贴/手动建卡
  → 卡片进入复习队列
```

## 4. 数据模型

### 4.1 核心实体

```swift
// 内容源：用户输入的原始内容
Source {
    id: UUID
    type: SourceType           // .text, .url
    raw_content: String        // 原始文本或 URL
    extracted_text: String     // 提取后的正文
    title: String?             // 内容标题
    domain: String?            // URL 来源域名
    created_at: Date
    tags: [String]             // AI 自动打的标签
}

// 卡片：AI 生成的复习卡片
Card {
    id: UUID
    source_id: UUID            // 关联的内容源
    question: String           // 正面：问题
    answer: String             // 背面：答案
    source_quote: String       // 原文引用

    // SM-2 算法状态
    repetition: Int            // 连续正确次数 (初始 0)
    ease_factor: Double        // 难度因子 (初始 2.5)
    interval: Int              // 复习间隔天数 (初始 0)
    next_review_date: Date     // 下次复习日期

    created_at: Date
    is_active: Bool            // 用户可以归档不想要的卡片
}

// 复习记录：每次复习的详细记录
ReviewLog {
    id: UUID
    card_id: UUID
    result: ReviewResult       // .remembered, .forgot
    reviewed_at: Date
}

// 每日状态：复习后的状态记录
DailyStatus {
    id: UUID
    date: Date                 // 唯一，每天只有一条
    energy_level: Double       // 0.0 - 10.0
    keyword: String?           // 可选关键词
    cards_reviewed: Int        // 今日复习卡片数
    cards_remembered: Int      // 今日记住的卡片数
    created_at: Date
}

// 订阅状态 (Phase 4)
Subscription {
    id: UUID
    status: SubscriptionStatus // .free, .trial, .pro, .expired
    trial_start: Date?         // 试用开始时间
    expires_at: Date?          // 订阅到期时间
}

// 提示展示记录 (Phase 4，用于频控)
PromptLog {
    id: UUID
    type: PromptType           // .subscription, .notification
    shown_at: Date             // 展示时间
    action: UserAction         // .tapped, .dismissed, .not_interested
}
```

### 4.2 数据关系

```
Source (1) ──→ (N) Card
Card   (1) ──→ (N) ReviewLog
日期    (1) ──→ (1) DailyStatus
App    (1) ──→ (1) Subscription      // 全局唯一
提示事件 ──→ (N) PromptLog            // 用于频控判断
```

## 6. UI/UX 设计规范

### 6.1 设计原则

1. **极简优先**：每个页面只做一件事
2. **手势驱动**：能滑就不点，减少认知负担
3. **温暖不说教**：语气像朋友，不像老师
4. **信息密度低**：留白充足，呼吸感强

### 6.2 配色方案

```
主色调：
- Primary:    #4A90D9 (柔和蓝，知性、平静)
- Secondary:  #F5A623 (暖橙，能量、活力)

背景：
- Background: #FAFBFC (极浅灰白)
- Card:       #FFFFFF (纯白卡片)
- Surface:    #F0F2F5 (次级背景)

文字：
- Title:      #1A1A2E (深墨)
- Body:       #4A4A68 (灰紫)
- Caption:    #8E8EA0 (浅灰)

语义色：
- Remembered: #4CAF50 (绿)
- Forgot:     #FF6B6B (红)
- Energy Low: #FF9800 (橙)
- Energy High:#4CAF50 (绿)
```

### 6.3 字体规范

```
- 大标题:   .largeTitle   (34pt, Bold)
- 标题:     .title2       (22pt, Bold)
- 卡片问题:  .title3       (20pt, Semibold)
- 卡片答案:  .body         (17pt, Regular)
- 说明文字:  .caption      (12pt, Regular)
```

## 11. 温和变现策略（不惹恼用户）

### 11.1 原则

1. 先交付价值，再谈付费：用户至少完成 1 次“导入→复习→回顾”闭环后才出现订阅提示。
2. 付费是增强，不是阉割：免费版保持完整基础体验（可导入、可复习、可看基础统计）。
3. 低频触达：同一用户 14 天内最多 1 次订阅提示，可手动关闭一段时间。
4. 语气克制：不制造焦虑，不使用“你正在落后”类文案。

### 11.2 建议套餐与定价

- Free（长期可用）：
  - 基础导入（文本/URL）
  - 基础卡片复习
  - 基础周统计
- Pro（订阅）：
  - 高级 AI 能力（卡片重写/难度优化/风格切换）
  - 深度洞察（跨周趋势、主题效率对比、个性化建议）
  - 历史回顾与导出（周报导出、长期轨迹）

**参考定价（需根据实际 AI 成本调整）：**

| 方案 | 价格 | 折算月价 | 说明 |
|------|------|---------|------|
| 月付 | ¥15/月 | ¥15 | 低门槛试水 |
| 年付 | ¥108/年 | ¥9 | 年付 40% 折扣，锁定长期用户 |

> 定价依据：月度 AI 成本约 $0.22/用户（≈ ¥1.6），月付 ¥15 对应 AI 成本占比约 11%，满足 ≤ 20% 的毛利目标。

### 11.3 付费提示触发时机（建议）

仅在以下任一条件满足后触发一次轻提示：
1. 连续 7 天中复习 ≥ 4 天；
2. 累计生成卡片 ≥ 40 张；
3. 第 3 次查看周报后。

### 11.4 触达频控与反感保护

1. 弹窗频控：14 天内最多 1 次。
2. 用户点击“不感兴趣”后：30 天不再展示。
3. 通知频控：每天最多 1 条；支持一键静音 7 天。
4. 节假日保护：默认降低提醒频率，避免过度打扰。

### 11.5 示例文案（温和语气）

- 标题：`想更快把内容变成可复习的行动卡片？`
- 正文：`如果你愿意，Pro 可以帮你自动优化卡片质量与周报洞察。免费功能仍可继续使用。`

## 12. 隐私与用户信任（商业化前置条件）

1. 首次导入时明确说明：哪些数据会发送给 AI 服务，目的是什么。
2. 提供设置项：允许用户关闭“上传内容用于 AI 洞察”。
